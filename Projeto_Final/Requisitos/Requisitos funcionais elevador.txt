-- Requisitos Funcionais do Sistema de Elevador
RF-01: O sistema deve realizar o controle de um conjunto de 3 elevadores;
RF-02: O sistema deve possuir 3 elevadores;
RF-03: O sistema deve possuir 15 andares;
RF-04: O sistema deve possuir 2 botões para chamada de elevador, para cada um deles, em cada andar;
	4.1: 1 botão caso o usuario queira subir de andar;
	4.2: 1 botão caso o usuario queira descer de andar;
RF-05: O sistema deve possuir 17 botões dentro de cada elevador;
	5.1: 1 - 15 botões com os andares desejados;
	5.1.1: Ao ser pressionado um botão de andar o elevador deve enviar uma solicitação de subida ou descida em direção ao andar;	
	5.1.2: Caso o sistema esteja subindo ele deve atender todas as solicitações de subida antes de processar uma de descida;	
	5.1.3: Caso o sistema esteja descendo ele deve atender todas as solicitações de descida antes de processar uma de subida;	
	5.1.4: Quando subindo o sistema deve atender a solicitação de andar acima mais próximo, independente da ordem de entrada das solicitações de subida;	
	5.1.5: Quando descendo o sistema deve atender a solicitação de andar abaixo mais próximo, independente da ordem de entrada das solicitações de descida;	
	5.2: 1 botão para abrir a porta do elevador;
	5.3: 1 botão para fechar a porta do elevador;
RF-06: Ao apertar o botão de abertura de porta o sistema deve verificar se o elevador está parado em um andar e se a porta está fechada, para então abrir a porta;
RF-07: Os elevadores não devem se mover com a porta aberta;
RF-08: Ao apertar o botão de fechamento de porta o sistema deve verificar se o elevador está parado em um andar e se a porta está aberta, para então solicitar o fechamento da porta;
RF-09: O elevador chamado deve ser independente de qual dos 3 sets de botão foi apertado;
RF-10: O sistema deve, ao ser apertado um dos botões de chamada do elevador disponivel em cada andar, chamar um elevador ao andar do botão, sendo suas prioridades de chamada:
	10.1:  Há um elevador parado no mesmo andar do botão;
	10.2:  Há um elevador em movimento com direção ao andar do botão;
	10.3:  O elevador parado mais perto do botão;
	10.4:  Caso todos os elevadores estejam indo em desencontro ao andar do botão ele chamará o primeiro elevador a parar sem outras solicitações ou com solicitações em direção ao andar do botão;
	10.4:  Em caso de empate o critério de prioridade é respectivamente a ordem de identificação direita,central,esquerda;
RF-11: Após chamado o elevador selecionado deve subir, ou descer, em direção ao andar indicado;
RF-12: Ao detectar que o elevador está na posiçao certa em relação ao botão do andar selecionado o elevador deve parar;
RF-13: Ao chegar no andar do botão apertado o elevador deve abrir a porta e esperar 20 segundos sem aceitar nova solicitações de chamada, a não ser que o botão de fechamento de porta tenha sido apertado;
RF-14: O sistema não deve fechar a porta do elevador caso haja algo em seu caminho;
RF-15: Caso haja algo no vão da porta o fechamento deve ser cancelado;
RF-16: Caso haja algo na frente da porta o sistema deve esperar que o obstaculo seja retirado antes de solicitar outro fechamento de porta;
RF-17: O sistema deve realizar a comunicação com o simulador através da porta serial;
RF-18: Caso um botão seja apertado ele deve permanecer aceso até sua ação correspondente ser concluida;

-- Requisitos Não Funcionais do Sistema de Elevador
RNF-01: O sistema deve ser realizado com o kit EK-TM4C1924XL;
RNF-02: O sistema deve fazer uso da biblioteca Drivelib;
RNF-03: O sistema deve fazer uso da API CMSIS-RTOS;
RNF-04: O sistema deve ser programado em C;
RNF-05: A IDE utilizada deve ser a IAR arm;
